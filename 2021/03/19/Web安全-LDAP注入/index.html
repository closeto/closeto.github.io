    LDAP协议简介

    LDAP（轻量级目录访问协议，Lightweight Directory Access Protocol）是一种应用层协议，用于访问和操作分目录访问协议。它提供了一种标准化的方法式，使客户端可以与目录服务器进入行通信，并针对目录中的信息进行查询、添加、修改和删除等操作。
    运行于TCP/IP之上的协议，端口号为：389， 加密636（SSL）

    目录树概念

    目录树： 在一个目录服务系统中，整个目录信息集可以表示为一个目录信息树，树中的每个节点是一个条目。
    条目： 每个条目就是一条记录，每个条目有自己的唯一可区别的名称（DN）。
    对象类： 与某个实体类型对应的一组属性，对象类是可以继承的，这样父类的必须属性也会被继承下来。
    属性： 描述条目的某个方面的信息，一个属性由一个属性类型和一个或多个属性值组成，属性有必须属性和非必须属性。

    DN：唯一标识一条记录的位置，类似于主键
    DC：域名的部分，其格式是将完整的域名分成几部分，如域名为example.com变成dc=example, dc=com
    OU：组织单位，组织单位可以包含其他各种对象，如一条记录（包括其他组织单元）
    CN：公共名称，即一条记录的名称
    Entry：条目记录数
    baby条目的DN就为cn=baby,ou=marketing,ou=pe=ple,dc=mydomain,dc=org``

    LDAP查询语法

    =：等于
    (cn=steve)：查询名为steve的人

    ：通配符
    (cn=s)：查询名字以s开头的人

    &：逻辑与
    (&(cn=s*)(sn=*d))：查询名字以s开头且以d结尾的人

    |：逻辑或
    (|(cn=s*)(cn=t*))：查询名字以s或t开头的人

    !：逻辑非
    (!cn=John)：查询名字不为John的人

    LDAP协议的应用

    1. 目录服务：LDAP最主要的应用是作为目录服务的基础。它被广泛用于存储和组织大量的结构化信息，如用户、组织、设备等的属性。LDAP提供了一种标准化的方式来访问和管理目录数据，使得用户和应用程序可以轻松地查询和更新目录中的信息。
    2. 身份认证和授权：LDAP用作身份认证和授权的基础。许多身份认证系统使用LDAP来存储和验证用户凭据，例如用户名和密码。LDAP的分层结构和查询能力使得身份认证系统可以有效地搜索和匹配用户凭据，并进行访问控制和授权管理。
    3. 用户管理：LDAP广泛用于用户管理和组织结构管理。组织可以使用LDAP来存储和管理用户账户、权限、组织单位等信息。通过LDAP，管理员可以进行用户的创建、删除、修改和查询操作，以及管理用户所属的组织结构关系。
    4. 地址簿和联系人管理：LDAP可用于实现企业内部的地址簿和联系人管理系统。LDAP目录可以存储和管理员工、合作伙伴、客户等联系人的信息，包括姓名、电话号码、电子邮件地址等。通过LDAP的查询功能，用户可以方便地搜索和获取所需的联系人信息。
    5. 协作和资源共享：LDAP可用于支持协作和资源共享。它可以存储和管理团队成员、权限、资源等信息，使得团队成员可以通过LDAP来访问和共享所需的资源，如共享文件夹、协作文档等。
    6. 单点登录（SSO）：LDAP常用于实现单点登录系统。通过LDAP作为身份认证和用户存储的后端，用户可以通过一次登录获得访问多个应用和服务的权限，提高用户体验和安全性。

    LDAP与AD的关系

    AD（Active Directory）是一种基于LDAP的目录服务，它使用LDAP作为其核心协议之一。

    AD域是建立在LDAP协议之上的一个更高级的目录服务，它扩展了LDAP协议，并添加了许多额外的功能和特性，如安全性、域控制器、组策略等。

    AD使用LDAP协议来实现以下功能
    ①目录访问：AD域作为一个分布式目录服务，使用LDAP协议来提供对目录中存储的数据的访问。客户端可以使用LDAP协议通过AD域控制器与AD域进行通信，执行查询、添加、修改和删除等操作。
    ②认证和授权：AD域使用LDAP协议进行用户身份认证和授权。客户端可以使用LDAP协议向AD域控制器发送身份认证请求，验证用户提供的凭据（如用户名和密码）。一旦用户身份验证成功，AD域可以使用LDAP协议提供对授权信息的访问，确定用户的权限和访问权限。
    ③目录结构和架构：AD域使用LDAP的数据模型和架构，基于条目（Entry）和属性（Attribute）来组织和描述目录中的数据。LDAP协议定义了AD域中数据的存储方式、属性类型和层次结构。
