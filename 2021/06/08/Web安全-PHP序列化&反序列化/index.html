    反序列化

    字符串——》对象 函数unserialize

    1、反序列化生成对象里边的值，由反序列化的值提供，与源对象中的值无关
    2、反序列化不会改变类的成员方法，需要调用方法后才能触发
    3、反序列话正确时，“ ;} ”是反序列化介素符，后边的字符串不会影响反序列化的结果

    反序列化——Phar

    ​ PHAR（”PHP Archive“）时PHP里类似JAR的一种打包文件。PHP > 5.3版本中默认开启且支持，文件以phar后缀结尾，类似压缩包文件有固定格式，文件中可以看到关键字[]、[序列化的对象]

    文件包含：phar伪协议、可读取phar文件

    漏洞利用原理：

    ​ 3、必须得有phar文件才能进行利用

    Phar利用条件：
    1、phar文件能上传到服务器端
    2、要有可用反序列化魔术方法做为跳板
    3、要有文件操作函数，file_exists()、fopen()、file_get_contents()
    4、文件操作函数参数可控，且:、/、phar等特殊字符没有被过滤

    phar反序列化防范

    1. 只从可信任的来源获取 PHAR 包，并确保它们是由可信任的开发人员创建的。
    2. 将 PHP 版本升级到最新的稳定版本，以便获得最新的安全更新。
    3. 禁用不必要的 PHP 函数和扩展，例如 eval() 函数和 system() 函数。
    4. 实施严格的输入验证和过滤，以确保不接受来自未经验证的用户的恶意输入。
    5. 在生产环境中尽可能减少使用 unserialize() 函数，或者使用更安全的替代方案，例如 json_decode() 函数。
    6. 在反序列化之前对输入进行类型检查，以确保输入是正确的对象类型。
    7. 配置 PHP 以限制 PHP 进程可以打开的文件和网络资源的数量，以减轻任意代码执行的影响。

    PHP伪协议

    文件包含

    本地文件使用 include 或 require 函数，

    远程文件使用 include_once、require_once 函数或者 file_get_contents() 函数等

    file://

    条件：

    • allow_url_fopen:off/on
    • allow_url_include :off/on

    作用：
    用于访问本地文件系统****，需要知道绝对路径****在CTF中通常用来读取本地文件的且不受allow_url_fopen与allow_url_include的影响。~~
    include()/require()/include_once()/require_once()参数可控的情况下，如导入为非.php文件，则仍按照php语法进行解析，这是include()函数所决定的。

    说明：
    file:// — 访问本地文件系统
    说明文件系统 是 PHP 使用的默认封装协议，展现了本地文件系统。 当指定了一个相对路径(不以://indows 盘符开头的路径)提供的路径将基于当前的工作目录。在很多情况下是脚本所在的目录，除非被修改了。 使用 CLI 的时候，目录默认是脚本被调用时所在的目录。在某些函数里，例如 fopen() 和 file_get_contents() ， include_path 会可选地搜索，也作为相对的路径。
    有时会出现file:///****** 的情况这个原因是访问了文件的根目录

    http://

    文件前缀加http或https

    条件：

    * allow_url_fopen:on
    * allow_url_include:on

    作用：
    常规 URL 形式，允许通过 HTTP 1.0 的 GET方法，以只读访问文件或资源。CTF中通常用于远程包含。

    zip&bzip2&zlib

    条件

    allow_url_fopen:off/on
    allow_url_include :off/on
    作用：zip:// & bzip2:// & zlib:// 均属于压缩流，可以访问压缩文件中的子文件，更重要的是不需要指定后缀名，可修改为任意后缀：jpg png gif xxx 等等。

    示例
    1、 zip://[压缩文件绝对路径]#压缩文件内的子文件名

    压缩 phpinfo.txt 为 phpinfo.zip ，压缩包重命名为 phpinfo.jpg ，并上传

    2、compress.bzip2://file.bz2
    压缩 phpinfo.txt 为 phpinfo.bz2 并上传(同样支持任意后缀名)

    3、compress.zlib://file.gz
    压缩 phpinfo.txt 为 phpinfo.gz 并上传(同样支持任意后缀名)

    data://

    条件

    allow_url_fopen:on
    allow_url_include :on
    作用
    自PHP>=5.2.0起，可以使用data://数据流封装器，以传递相应格式的数据。通常可以用来执行PHP代码。

    用法
    data://text/plain,
    data://text/plain;base64,

    示例

    data://text/plain,

    data://text/plain;base64,

    phar://

    phar://协议与zip://类似，同样可以访问zip格式压缩包内容，在这里只给出一个示例

    http://127.0.0.1/include.php?file=phar://E:/phpStudy/PHPTutorial/WWW/phpinfo.zip/phpinfo.txt
    注意这里与zip://不同的地方是，phar访问压缩包内容是通过/访问，而zip是通过#访问
