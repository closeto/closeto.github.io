    📌搭建本地演示环境中遇到主从DNS服务器搭建，故作此记录。

    文件介绍

    1、bind配置文件&资源区域错误排查手段：
    ​ named-checkconf检查配置文件、named-checkzone 域名 /var/named/named.xxx资源解析文件

    2、bind服务文件介绍：

    ​ /etc/named.conf 主配置文件
    ​ /etc/named.rfc1912.zones 区域文件（可以存在多个）
    ​ /var/named/ 资源区域文件
    ​ /var/named/data/ 存放区域文件的目录。用于定义域名对应的内网IP和配置

    4、静态域和动态域的区分：

    ​ 静态域：/etc/named.区域文件中allow_update{none;};
    ​ 动态域：/etc/named.区域文件中allow_update{ip_server;};

    named安装

    1、安装Dns-server服务（安装后文件的属性为640，属组为named，id named查看）
    ​ yum install -y bind

    2、安装后查看服务是否为”enable“
    ​ systemctl list-unit-files | grep named

    4、修改/etc/resolv.conf，新增如下配置（nameserver 本机IP）
    search 域名 #配置后ping主机名自动会加上域名
    nameserver x.x.x.x修改后重启生效–》在网卡配置中新增：
    ​ DNS1=x.x.x.x #dns服务器地址
    ​ PEERDNS=no #不自动获取dns地址（禁止获取dhcp分配的dns）

    5、修改hostname为NS记录的名称，一般公网解析会用到，若是内网搭建dns服务器则忽略
    ​ 例如：dns1.cto.com

    6、区域配置文件zonefile，注意事项：
    ​ a.宏定义（以$开头）
    ​ $TTL 60
    ​ $ORIGIN
    ​ b.注释（以；开头）

    7、DNS主辅同步配置：
    ​ a.DNS的版本必须小于等于主
    ​ b.主服务器named.conf中option字段里，配置：allow-transfer{ip;};、also-notify{ip;};
    ​ c.辅服务器named.conf中option字段里，配置：masterfile-format text; #没有这个参数从主服务器传递过来的资源记录为乱码
    ​ d.辅服务器DNS的zone文件，type slave；masters {主服务器IP;};
    ​ e.启动辅服务器前，检查完全区域传送：dig -t AXFR 域名 @主服务器ip #zone文件中file字段需改写“/var/named/slaves/zone_file”
    ​ f.辅服务器禁止修改DNS数据库文件

    查看named启动过程（故障排查）
    named -g

    DNS维护

    nsupdate

    DNS工具安装：yum install -y bind-utils
    ​nsupdate使用
    ​ >server DNS_SERVER_IP
    ​ >update add/del cto.cto.com 60 A 1..2.3.4 #主机名.域名 TTL type IP_ADDR
    ​ >send
    ​ >
    ​ dig -t A cto.cto.com @DNS_server_IP +short
    ​ 使用nsupdate新增记录，会新增文件/var/named/资源区域.jnl
    ​ 不会写入到资源区域文件，会产生动态文件xxx.jnl，不利于维护，不使用

    静态域和动态域维护

    ​静态域：/etc/named/zones下，无allow-update {dns服务器IP;};
    ​ 主辅DNS同步：手动更新-然后-重启named
    ​ 1、手动设置主服务器/var/named/named*.zones文件中的serial，serial value+1
    ​ 2、设置主/辅服务器/var/named/named*.zones文件中的refresh值，该值决定于多久重新从主DNS服务器获取资源记录
    ​动态域：/etc/named/zones下，有allow-update {dns服务器IP;};
    ​ 主辅DNS同步：自动更新，不需要重启named
    ​ nsupdate，该命令增加的资源记录会在生成/var/namd/named.xxx.zone.jil文件，该文件内容不可读
    ​ >server DNS_SERVER_IP
    ​ >update add/del cto.cto.com 60 A 1..2.3.4 #主机名.域名 TTL type IP_ADDR
    ​ >send
    ​ >quit
    ​ dig -t A cto.cto.com @DNS_server_IP +short

    mdc远程管理

    配置rndc远程管理步骤–静态域

    1、在服务端生成rndc-key （执行 rndc-confgen -r /dev/urandom）

    2、将生成的信息放在对应的文件目录中，删除/etc/rndc.key
    ​ End of named.conf中间的信息放在/etc/named.conf中option以外的位置，主DNS服务器配置即可，备DNS不用
    ​ server_ip：填写主DNS服务器IP
    ​ allow_ip ：填写主备DNS服务器IP
    ​ End of rndc.conf中间的信息放在创建的/etc/rndc.conf中，主备DNS服务器都需要
    ​ default_server：填写主DNS服务器的IP地址

    3、使用 rndc status查看服务状态是否running
    ​ rndc reload 域名 #获取域名更新（回显为zone reload queued表示更新成功，仅备机执行主机执行会报错）

    4、主DNS在/var/named/下资源区域文件中添加解析规则后，变更serial的值

    5、备机上执行：
    ​ rndc reload 域名
    ​ dig -t A cto.cto.com @DNS_server/slave_IP +short #查看是否更新成功（主DNS不用重启named进程）

    配置rndc远程管理步骤–动态域

    1、在服务端生成rndc-key （执行 rndc-confgen -r /dev/urandom）

    2、将生成的信息放在对应的文件目录中，删除/etc/rndc.key
    ​End of named.conf中间的信息放在/etc/named.conf中option以外的位置，主DNS服务器配置即可，备DNS不用
    ​ server_ip：填写主DNS服务器IP
    ​ allow_ip ：填写主备DNS服务器IP
    ​End of rndc.conf中间的信息放在创建的/etc/rndc.conf中，主备DNS服务器都需要
    ​ default_server：填写主DNS服务器的IP地址

    3、使用 rndc status查看服务状态是否running
    ​ rndc reload 域名 #获取域名更新（回显为zone reload queued表示更新成功，仅备机执行主机执行会报错）

    4、主DNS在/var/named/下资源区域文件中添加解析规则后，变更serial的值

    5、rndc freeze 域名 #先冻结域

    6、rndc thaw 域名 #解冻域名

    7、nsupdate 添加资源记录
