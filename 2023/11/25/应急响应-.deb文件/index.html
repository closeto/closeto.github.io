    0x00 .deb文件

    Ⅰ、ar文件介绍

    ​ .deb仅仅是文件的后缀(主要用于区分操作系统)，实际上.deb使用Unix的 ar（Archive）格式。ar文件格式为归档文件，主要包含debian-binary、control.tar.xxx、data.tar.xxx这三个文件，后两个为压缩文件。

    下列所示为Archive文件格式(也就是.deb)：

    :heavy_exclamation_mark:注意：fileMode格式 1 xxxx zzzz 644
    1 =文件类型（0=普通文件，1=目录）
    xxxx=所属者
    zzzz =所属组
    644 =文件权限

    Ⅱ、deb文件介绍

    ​ .deb 文件是 Debian 软件包的文件格式，主要用于 Debian 及其派生系统。Debian 是一个流行的 Linux 发行版，而其派生系统包括 Ubuntu、Linux Mint 等。因此，.deb 文件通常用于这些基于 Debian 的 Linux 发行版。:triangular_flag_on_post:Kylin、UOS也使用的是.deb文件

    组成（4部分） 文件格式 详细
    文件头 Magic Number 16进制数：21 3C 61 72 63 68 3E 0A
    二进制数据 debian-binary 包含文件头、版本号、文件属性&权限等
    软件控制信息 control.tar.xxx 包含软件包的名称、版本、架构、维护者信息、依赖关系等
    安装数据 data.tar.xxx 包含软件包的二进制可执行文件、库文件、配置文件等，即软件包的实际内容。

    例如data.tar.xxx文件中的xxx为tar的压缩算法，大部分为zst即Zstandard 压缩

    :black_nib:将AR字段变更为.deb，得到树状图格式如下：deb包中的文件目录在安装后放在根对应目录

    preinst文件：可以停止作用于待升级软件包的服务，直到软件包安装或升级完成。
    postinst文件：软件安装完后，执行该Shell脚本，一般用来配置软件执行环境，必须以“#!/bin/sh”为首行。
    prerm 文件：该脚本负责停止与软件包相关联的daemon服务。它在删除软件包关联文件之前执行。
    postrm文件：软件卸载后，执行该Shell脚本，一般作为清理收尾工作，必须以“#!/bin/sh”为首行。

    Ⅳ、changelog

    :star:changelog标准格式如下（必须）

    Changelog 文件字段解释：

    • my-package： 软件包的名称。
    • (1.0-1)： 软件包的版本号，包括 Debian 版本号和修订号。
    • unstable： 软件包所属的 Debian 发行版（可以是 stable、testing、unstable 等）。
    • urgency=low： 修改的紧急程度，可以是 low、medium、high。
    • \* Initial release.： 对该版本的简要说明，说明这个版本的主要变化。
    • -- Your Name <your.email@example.com> Sat, 01 Jan 2023 12:00:00 +0000： 修改记录的作者、日期和时间。

    Ⅴ、copyright

    :star:copyright标准格式如下（可选）

    • Format: https://www.examp.org/doc： 指定了 copyright 文件的格式，通常是指向 Debian 打包政策中关于版权格式的文档链接。
    • Files: \*： 表示以下版权和许可信息适用于软件包中的所有文件。
      • Copyright: 2023 Your Name <your.email@example.com>： 说明软件包的版权归属于哪个作者或团队，以及年份和联系信息。
      • License: GPL-3+： 说明软件包的许可证类型和版本。在这个示例中，使用的是 GNU General Public License（GPL）的版本 3 或更高版本。
    • Files: debian/\*： 表示以下版权和许可信息适用于软件包的 debian 目录及其内容，通常这包括打包脚本等。
      • Copyright: 2023 Your Name <your.email@example.com>： 说明 debian 目录下的文件的版权信息。
      • License: GPL-3+： 同样，说明 debian 目录下的文件采用的许可证。
    • License: GPL-3+： 提供了一个许可证段落，详细说明了使用的许可证类型和版本。

    Ⅵ、triggers

    用于在软件包的安装或卸载过程中触发其他动作
    /usr/share/my-demo 中的文件或目录发生变化，就会触发与这个路径相关的触发器。

    Ⅶ、desktop

    1. [Desktop Entry]：
      • 这是 .desktop 文件的必需部分，表示一个桌面文件。
    2. Name=My Demo：
      • 显示在桌面环境中的应用程序名称。
    3. Comment=A simple demo application：
      • 提供关于应用程序的简短描述或注释。
    4. Exec=/usr/bin/my-demo：
      • 指定启动应用程序的命令。这个字段应该包含应用程序的可执行文件路径。
    5. Icon=/usr/share/icons/my-demo.png：
      • 指定应用程序的图标文件的路径。这个字段应该包含一个图标文件的路径，通常是一个 .png 或 .svg 文件。
    6. Type=Application：
      • 指定 .desktop 文件描述的条目的类型。在这里，它是一个应用程序。
    7. Categories=Utility;Development;：
      • 指定应用程序所属的类别。这些类别用于组织菜单。在这个例子中，应用程序被归类为“Utility（实用工具）”和“Development（开发）”。
    8. Terminal=false：
      • 表示应用程序是否在终端中运行。如果设置为 true，则表示应用程序需要在终端中运行。
    9. StartupNotify=true：
      • 指示桌面环境是否应该在应用程序启动时显示启动通知。如果设置为 true，则表示桌面环境将显示一个通知，告知用户应用程序已经启动。
    10. MimeType=text/plain;：
      • 指定与应用程序相关联的 MIME 类型。在这个例子中，它表示应用程序能够处理纯文本文件。
    11. Actions=Open;Edit;：
      • 指定桌面动作的列表。这里定义了两个动作：Open 和 Edit。
    12. [Desktop Action Open]：
      • 这是动作 Open 的开始。这个部分包含与打开文件相关的信息。
    13. Name=Open File：
      • 显示在菜单中的动作名称。
    14. Exec=my-demo --open-file：
      • 指定执行动作的命令。
    15. Icon=accessories-text-editor：
      • 指定动作的图标。
    16. OnlyShowIn=Unity;：
      • 指定只在 Unity 桌面环境中显示这个动作。
    17. [Desktop Action Edit]：
      • 这是动作 Edit 的开始。这个部分包含与编辑文件相关的信息。
    18. Name=Edit File：
      • 显示在菜单中的动作名称。
    19. Exec=my-demo --edit-file：
      • 指定执行动作的命令。
    20. Icon=accessories-text-editor：
      • 指定动作的图标。
    21. OnlyShowIn=Unity;：
      • 指定只在 Unity 桌面环境中显示这个动作。

    可选值：

    • Type 的可选值：
      • Application：表示一个应用程序。
      • Link：表示一个链接。
      • Directory：表示一个目录。
    • Categories 的可选值：
      • Audio：音频应用程序。
      • Video：视频应用程序。
      • Development：开发工具。
      • Games：游戏。
      • Graphics：图形应用程序。
      • Network：网络应用程序。
      • Office：办公套件。
      • System：系统工具

    Ⅷ、man

    :pushpin:存放系统手册的目录，如man ls，目录解释如下：

    man1：用户命令的 man 页面。
    man2：系统调用和内核服务的 man 页面。
    man3：库函数的 man 页面。
    man4：设备文件和驱动程序的 man 页面。
    man5：配置文件和文件格式的 man 页面。
    man6：游戏的 man 页面。
    man7：杂项的 man 页面，例如协议、文件格式等。
    man8：系统管理员命令的 man 页面。

    • .TH 行定义了 man 页面的标题和相关信息。
    • .SH NAME 和 .SH SYNOPSIS 部分定义了命令的名称和用法。
    • .SH DESCRIPTION 部分提供了关于命令的描述。
    • .SH OPTIONS 部分列出了命令的选项。
    • .SH EXAMPLES 部分包含了一些使用示例。
    • .SH SEE ALSO 部分提供了其他相关的 man 页面。
    • .SH AUTHOR 部分提供了命令的作者信息。
