    Windows端口转发

    作用：将本地指定端口的流量转发到目标端口上，常用于内网环境

    前提条件：

    ​ 1、使用管理员权限或者提权

    ​ 2、关闭防火墙或设置防火墙规则

    ​ 3、仅适用于TCP端口，且主机支持Ipv6（在没在使用无所谓）

    1. 新增端口转发：

    netsh interface portproxy add v4tov4 listenaddress=监听地址 listenport=监听端口 connectaddress=转发IP connectport=转发端口

    netsh interface portproxy add v4tov4 listenaddress=192.168.. listenport=* connectaddress=192.168.64.12 connectport=8443（不建议使用）

    netsh interface portproxy add v4tov4 listenport=9988 connectaddress=192.168.88.110 connectport=9988

    2. 查看转发了哪些端口

    netsh interface portproxy show all

    3. 删除转发规则

    netsh interface portproxy delete v4tov4 listenaddress=内网IP listenport=本机外部端口

    PS：自己添加的防火墙规则没有“组”的信息

    Linux端口转发

    作用：将本地指定端口的流量转发到目标端口上，常用于内网环境

    PS：以Centos7举例，7.0之后使用firewall-cmd启动防火墙

    前提条件：

    ​ 1、需要启动防火墙，仅限centos7之后使用

    ​ 2、打开防火墙或设置防火墙进入规则

    ​ 3、开启伪装IP，不开启不行

    步骤：

    开启伪装IP： firewall-cmd –permanent –add-masquerade

    配置端口转发：firewall-cmd –permanent –add-forward-port=port=本地端口:proto=协议:toaddr=转发目的IP:toport=目的端口

    删除转发规则：firewall-cmd –remove-forward-port=port=本地端口:proto=协议:toport=目的端口:toaddr=转发目的IP –permanent

    更新防火墙规则：firewall-cmd –reload

    可能的错误：

    防火墙服务的状态：systemctl status firewalld.service

    启动防火墙：systemctl start firewalld.servic

      或者  *systemctl start firewalld*

    关闭防火墙：systemctl stop firewalld.servic

      或者  *systemctl stop firewalld*

    获取所有支持的服务：firewall-cmd –get-service

    查看防火墙列表：firewall-cmd –list-all

      或者  firewall-cmd --list-all --zone public

    更新防火墙规则：firewall-cmd –reload
