    在分析某厂商检测设备时发现有使用YARA，由是再次做次记录

    **官方参考文档**：https://yara.readthedocs.io/en/stable/writingrules.html

    YARA的规则集由三部分组成：**规则头、元数据和规则体**
    规则头包括了规则名称和规则描述等信息
    元数据是可选项，可以提供一些额外的信息
    规则体由多个条件组成的,可以是文件特征、字符串、哈希值等

    YARA可以使用各种编程语言进行编写，例如C、Python等，可以在各种操作系统平台上使用，例如Windows、Linux等。

    YARA主要应用场景：
    文件检测
    进程检测
    恶意代码家族检测
    流量检测
    PS：AV、IPS、EDR等有使用，语法类似C/C++

    通配符(占位符)

    可以匹配一字节 或 半字节，匹配如下字符串：
    E2 34 11 C8 A1 FB E2 34 22 C8 A2 FB E2 34 33 C8 A3 FB

    取非

    波形符 (~) 为前缀的字节，它是非运算符。

    跳转符

    当定义内容可能变化但您知道变量块的长度的字符串时，使用跳转符。
    任何跳转 [XY] 都必须满足条件 0 <= X <= Y ；
    从 YARA 2.0 开始，X 和 Y 值没有限制

    文本字符串

    文本字符串还可以包含 C 语言中可用的转义序列的以下子集：

    \" 双引号
    \\ 反斜杠
    \r 回车符
    \t 水平制表符
    \n 新队
    \xdd 任何十六进制表示的字节

    字符串常用操作汇总：:small_red_triangle_down:

    关键词 字符串类型 概括 限制
    nocase 文本、正则表达式 忽略大小写 不能与xor、base64、 或 一起使用base64wide
    wide 文本、正则表达式 通过交错空 (0x00) 字符来模拟 UTF16 没有任何
    ascii 文本、正则表达式 也匹配 ASCII 字符，仅在wide使用时才需要 没有任何
    xor 文本 与单字节键异或文本字符串 不能与nocase、base64、 或 一起使用base64wide
    base64 文本 转换为3个base64编码的字符串 不能与nocase、xor、 或 一起使用fullword
    base64wide 文本 转换为 3 个 base64 编码的字符串，然后交错空字符，例如wide 不能与nocase、xor、 或 一起使用fullword
    fullword 文本、正则表达式 匹配项前面或后面没有字母数字字符 不能与base64或 一起使用base64wide
    private 十六进制、文本、正则表达式 匹配从未包含在输出中

    不区分大小

    **nocase**在同一行中的字符串定义末尾附加修饰符将字符串转换为不区分大小写模式

    字符串foobar将匹配Foobar、FOOBAR和fOoBaR

    执行程序入口点:star2:

    版本受限，可能无法使用

    entrypoint如果文件是可移植可执行文件 (PE) 或可执行可链接格式 (ELF)，则此变量将保存可执行文件入口点的原始偏移量，以防我们扫描文件

    如果我们正在扫描正在运行的进程，入口点将保存主可执行文件入口点的虚拟地址

    使用:在入口点查找某种模式以检测加壳程序或简单文件感染程序

    entrypoint规则中变量的存在意味着只有 PE 或 ELF 文件可以满足该规则。如果文件不是 PE 或 ELF，则任何使用此变量的规则都会计算为 false。

    内置函数

    1. pe:
      • 功能：用于处理和分析 PE (Portable Executable) 格式的文件，通常是 Windows 可执行文件。
      • 使用方式：在 YARA 规则中添加 import "pe"。
    2. elf:
      • 功能：提供 ELF (Executable and Linkable Format) 文件格式相关的功能，用于匹配和分析 Linux 和其他类 Unix 系统的可执行文件。
      • 使用方式：在 YARA 规则中添加 import "elf"。
    3. math:
      • 功能：提供了一些数学函数，例如计算平均值、标准差等。
      • 使用方式：在 YARA 规则中添加 import "math"。
    4. cuckoo:
      • 功能：用于与 Cuckoo Sandbox 集成，提供与威胁情报相关的功能。
      • 使用方式：在 YARA 规则中添加 import "cuckoo"。
    5. magic:
      • 功能：提供了与文件魔术数字（magic number）匹配相关的功能，用于根据文件头部信息识别文件类型。
      • 使用方式：在 YARA 规则中添加 import "magic"。
    6. dotnet:
      • 功能：用于匹配和分析 .NET 文件，包括 C# 编写的程序集。
      • 使用方式：在 YARA 规则中添加 import "dotnet"。
    7. hash:
      • 功能：提供了各种哈希函数，如 MD5、SHA-1、SHA-256 等，用于计算文件的哈希值。
      • 使用方式：在 YARA 规则中添加 import "hash"。
    8. time:
      • 功能：提供了与时间相关的功能，例如获取当前时间、时间戳等。
      • 使用方式：在 YARA 规则中添加 import "time"。
