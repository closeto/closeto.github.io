    二、内核编译-步骤

    1、进入Linux内核文件目录，设置环境
    export ARCH=x86

    2、配置board config,此处配置为 x86_64_defconfig
    make x86_64_defconfig

    3、make menuconfig（最后save）
    General setup —>
    —-> [*] Initial RAM filesystem and RAM disk (initramfs/initrd) support

    Device Drivers —>
    []Block devices —>
    [*]RAM block device support 4096改为65536

    4、进行内核编译
    make
    编译成功后的内核位置：arch/x86_64/boot/bzImage

    四、根文件系统镜像文件

    0、删除临时文件
    rm -rf rootfs.ext3
    rm -rf fs

    1.先制作一个空的镜像文件；
    dd if=/dev/zero of=./rootfs.ext3 bs=1M count=32

    2.然后把此镜像文件格式化为ext3格式；
    mkfs.ext3 rootfs.ext3

    3.创建挂载点，然后把此镜像文件挂载，并把根文件系统复制到挂载目录；
    mkdir fs
    mount -o loop rootfs.ext3 ./fs
    cp -rf ./_install/* ./fs

    4.卸载该镜像文件。
    umount ./fs

    5.打成gzip包。
    gzip –best -c rootfs.ext3 > rootfs.img.gz

    Linux-Kernel目录结构（简列）

    根目录 描述
    script 没和编译脚本
    security 安全相关代码
    arch 和具体cpu有关的代码，例如arm/mach-omap1，即ti公司的soc，此soc的cpu核是ARM提供
    init 内核启动代码
    drivers 驱动框架代码，例如i2c，dma， leds
    fs 文件系统
    kernel 注意和arm/kernel目录的区别。 kernel目录会调用arm/kernel。
    net 网络协议
    include 内核头文件
    mm 内存&内存管理相关
    usr 测试代码
    block 块设备代码
    ipc 进程通信
    crypto 加解密
    lib 内核通用库
