    Tomcat

    ①判断是否安装有tomcat相关服务

    ​ systemctl status tomcat.service 或者 systemctl status tomcat

    ②安装Tomcat

    ​ yum install -y tomcat //主程序

    ​ yum install tomcat-webapps tomcat-admin-webapps //默认浏览器管理界面

    ③启动Tomcat、判断Tomcat的运行状态

    ​ systemctl start tomcat //启动Tomcat程序

    ​ systemctl status tomcat //查看Tomcat启动后的状态

    ④关闭防火墙（可选）

    ​ systemctl stop firewalld //临时关闭防火墙

    ​ systemctl status firewalld //查看防火墙状态

    ​ systemctl disable firewalld //禁止开机自启防火墙

    ​ 默认访问界面：IP:8080

    生成SSL证书

    方法一
    yum install mod_ssl
    yum install openssl openssl-devel

    创建私钥
    openssl genrsa -out server.key 2048

    创建证书申请文件
    openssl req -new -key server.key -out server.csr

    创建有效时间为10年的自签名证书
    openssl x509 -req -in server.csr -out server.crt -signkey server.key -days 3650

    方法二
    创建时不加密码
    openssl genrsa -out server.key 2048
    openssl req -new -x509 -days 3650 -key server.key -out server.crt -subject

    【修改自建证书】
    vi /etc/httpd/conf.d/ssl.conf 修改途中的证书和私钥

    Nginx

    证书生成

    官网 http://nginx.org/download/
    Country Name 服务器/客户端证书与CA必须相同
    Organization Name 服务器/客户端证书与CA必须相同*
    Common Name (eg, your name or your server’s hostname) []:域名或IP地址，服务器/客户端证书与CA不能相同

    .key文件：密钥文件，SSL证书的私钥就包含在其中；

    .csr文件：这个文件里面包含着证书的公钥和其他一些公司信息，通过请求签名之后就可以直接生出证书；

    .crt文件：该文件中也包含了证书的公钥、签名信息以及根据不同类型证书携带不同的认证信息，如IP等（该文件在有些机构、系统中也可能表现为.cert后缀）；

    .pem文件：该文件相对比较少见，里面包含着证书的私钥以及部分证书信息。

    der,cer文件一般是二进制格式的，只放证书，不含私钥

    crt文件可能是二进制的，也可能是文本格式的，应该以文本格式居多，功能同der/cer

    pem文件一般是文本格式的，可以放证书或者私钥，或者两者都有

    pem如果只含私钥的话，一般用.key扩展名，而且可以有密码保护

    pfx,p12文件是二进制格式，同时含私钥和证书，通常有保护密码

    SSL单向认证

    【单向认证-仅验证服务器是否正规/合法】

    yum -y install gcc-c++ pcre-devel zlib-devel openssl-devel

    wget http://nginx.org/download/nginx-1.9.9.tar.gz

    tar zxvf nginx-1.9.9.tar.gz

    cd nginx-1.9.9

    ./configure –prefix=/usr/local/nginx –with-http_ssl_module

    make && make install

    生成服务端私钥

    openssl genrsa -out server.key 1024

    生成服务端公钥证书

    openssl req -new -x509 -key server.key -out server.crt -days 3650

    配置文件中增加

    ​ ssl_certificate ../ssl/server.crt; # server公钥证书

    ​ ssl_certificate_key ../ssl/server.key; # server私钥

    启动nginx

    ​ nginx

    ​ nginx -s reload

    ​ nginx -s stop

    双向认证-单个

    【双向认证-服务器和客户端互相认证对方是否正规/合法】

    服务器端密钥生成同前，配置文件中只需要加入客户端密钥

    单个客户端双向校验

    【client密钥生成】

    # 生成客户端证书私钥

    openssl genrsa -out client.key 1024

    # 生成客户端公钥证书

    openssl req -new -x509 -key client.key -out client.crt -days 3650

    # 生客户端p12格式证书，需要输入一个密码，选一个好记的，比如123456

    openssl pkcs12 -export -clcerts -in client.crt -inkey client.key -out client.p12

    ​ client公私钥两个文件合成得到一个p12文件，p12文件主要用于客户端

    config配置文件增加

    ​ ssl_client_certificate ../clientcrt/clientA.crt; # client公钥证书

    ​ ssl_verify_client on; # 开启客户端证书验证
