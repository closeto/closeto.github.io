    最短路径优先协议——OSPFv3

    数字通信 2019
    发布时间 : 2019-02-10 00:00

    IPv6中使用地址/前缀长度描述地址，而没有子网掩码的概念。IPv6中的链路（Link）就类似于IPv4中子网或网络

    OSPFv3使用链路本地地址建立邻居关系，即使接口全球单播地址发生变动也不会影响邻居建立

    IPv6地址扩大到128位后导致LSA长度增加

    链路本地地址（Link-Local）发送报文，Vlink除外（使用全局单播地址）

    接口可以配置多个全球单播地址，运行于每个Link进行通讯，不再基于subnet（基于链路本地地址运行）

    使用IPv6验证扩展头进行报文认证和加密

    OSPF v2与v3同异

    OSPFv2/v3相同点：网络/接口类型、接口/邻居状态机、LSDB、泛洪机制、协议报文、路由计算

    OSPFv3与v2不同点：

    ​ 基于链路运行：v2中建立邻居必须在同网段，v3中链路可以划分多个子网，只要链路地址相同既能建立邻居

    ​ 使用本地链路地址建立邻居：链路本地地址只针对本地链路有效，只能出现在Link-LSA中。建立Vlink必须使用全球单播地址

    ​ 链路支持实例复用：同一链路运行多个实例。物理接口实例instance-ID相同即可：ospfv3-进程号-区域-实例ID

    ​ 认证变化：OSPF-header自身没有认证字段，认证由扩展字段提供

    ​ stub区域支持：v2丢弃未知类型LSA，v3中未知LSA泛洪范围或链路U比特没有置位时，未知LSA才能向Stub区域泛洪

    ​ 报文不同：Option字段不同、LSA类型和内容不同

    报文格式

    OSPFv3报头格式

    OSPFv3用IPv6报文直接封装协议报文，协议号为89。OSPFv3分为5种报文：Hello报文、DD报文、LSR报文、LSU报文和LSAck报文。

    报文头格式，OSPFv3这五种报文具有相同的报文头格式，长度为24字节

    ![ospf 1.png](./images/ospf 1.png)

    字段 长度 含义
    Version 8比特 OSPF的版本号。对于OSPFv3，其值为3。
    Type 8比特 OSPFv3报文的类型：1：Hello报文。2：DD报文。3：LSR报文。4：LSU报文。5：LSAck报文。
    Packet length 16比特 OSPFv3报文的总长度，包括报文头在内，单位为字节。
    Router ID 32比特 发送该报文的ME设备标识。
    Area ID 32比特 发送该报文的ME设备的所属区域。
    Checksum 16比特 包含除了认证字段的整个报文的校验和
    Instance ID 8比特 OSPFv3多实例ID。
    0 8比特 预留值。

    Hello

    其作用为建立和维护邻接关系，周期性的在使能了OSPFv3的接口上发送。报文内容包括一些定时器的数值、DR、BDR以及已知的邻居

    ![ospf 1.png](./images/ospf 1.png)

    字段 长度 解释
    Interface ID 32比特 发送Hello报文的接口ID。
    Rtr Priority 8比特 DR优先级。默认为1(=0，则不能参与DR或BDR选举。高优先)
    Options 24比特 可选项：E：允许Flood AS-External-LSAs。MC：转发IP组播报文。N/P：处理Type-7 LSAs。DC：处理按需链路。
    HelloInterval 16比特 发送Hello报文的时间间隔。
    RouterDeadInterval 16比特 失效时间。如果在此时间内未收到邻居发来的Hello报文，则认为邻居失效
    Designated Router ID 32比特 DR的接口地址。
    Backup Designated Router ID 32比特 BDR的接口地址。
    Neighbor ID 32比特 邻居，以Router ID标识

    Hello报文在不同网络中的发送地址类型、发送时间间隔类型和时间间隔缺省值不同

    网络类型 发送地址类型 发送时间间隔类型 时间间隔缺省值
    广播 组播地址 HelloInterval 缺省情况下，接口发送Hello报文的时间间隔的值为10秒。
    NBMA 单播地址 DR、BDR、有能力成为DR的ME设备时，发送时间间隔为HelloInterval。邻居的状态为Down时，发送时间间隔为Pollinterval，其它情况为HelloInterval。 缺省情况下，接口发送Hello报文的时间间隔的值为30秒。缺省情况下，接口发送Pollinterval间隔为120秒。
    P2P 组播地址 HelloInterval 缺省情况下，接口发送Hello报文的时间间隔的值为10秒。
    P2MP 组播地址 HelloInterval 缺省情况下，接口发送Hello报文的时间间隔的值为30秒。

    ff02::5/6

    在同一网段上，其HelloInterval、PollInterval和RouterDeadInterval必须分别一致，否则不能形成邻居关系。在NBMA网络上还有一种PollInterval。

    DD

    两台ME设备在邻接关系初始化时，用DD报文描述本端ME设备的LSDB，进行数据库的同步。报文内容包括LSDB中每一条LSA的Header（LSA的Header可以唯一标识一条LSA）。LSA Header只占一条LSA的整个数据量的一小部分，这样可以减少ME设备之间的协议报文流量，对端ME设备根据LSA Header就可以判断出是否已有这条LSA。在两台ME设备交换DD报文的过程中，一台为Master，另一台为Slave。由Master规定起始序列号，每发送一个DD报文序列号加1，Slave方使用Master的序列号作为确认

    ![ospf 1.png](./images/ospf 1.png)

    字段 长度 解释
    Options 24比特 可选项：E：允许Flood AS-External-LSAs。MC：转发IP组播报文。N/P：处理Type-7 LSAs。DC：处理按需链路。
    Interface MTU 16比特 在不分片的情况下，此接口最大可发出的IP报文长度。
    I 1比特 当发送连续多个DD报文时，如果这是第一个DD报文，则置为1，否则置为0。
    M（More） 1比特 当发送连续多个DD报文时，如果这是最后一个DD报文，则置为0，否则置为1，表示后面还有其他的DD报文。
    M/S（Master/Slave） 1比特 当两台OSPFv3设备交换DD报文时，首先需要确定双方的主从关系，Router ID大的一方会成为Master。当值为1时表示发送方为Master。
    DD sequence number 32比特 DD报文序列号。主从双方利用序列号来保证DD报文传输的可靠性和完整性。
    LSA Headers – 该DD报文中所包含的LSA的头部信息。

    LSR

    两台ME设备互相交换过DD报文之后，需要发送LSR报文向对方请求更新LSA，内容包括所需要的LSA的摘要信息

    ![ospf 1.png](./images/ospf 1.png)

    字段 长度 解释
    LS type 16比特 LSA的类型号
    Link State ID 32比特 与LS Type一起描述路由域中唯一一个LSA。
    Advertising Router 32比特 产生此LSA的ME设备的Router ID。

    LS type、Link State ID和Advertising Router可以唯一标识出一个LSA。当两个LSA一样时，需要根据LSA中的LS sequence number、LS checksum和LS age来判断出所需要LSA的新旧。

    LSU

    LSU报文用来向对端ME设备发送其所需要的LSA或者泛洪本端更新的LSA，内容是多条LSA（全部内容）的集合。LSU报文在支持组播和广播的网络中是以组播形式将LSA泛洪出去。为了实现Flooding的可靠性传输，需要LSAck报文对其进行确认，对没有收到确认报文的LSA进行重传，重传的LSA是直接发送到邻居的

    ![ospf 1.png](./images/ospf 1.png)

    字段名 长度 含义
    Number of LSAs 32比特 LSA的数量。

    LS-ACK

    LSAck报文用来对接收到的LSU报文进行确认，内容是需要确认的LSA的Header（一个LSAck报文可对多个LSA进行确认）。LSAck报文根据不同的链路以单播或组播的形式发送，

    ![ospf 1.png](./images/ospf 1.png)

    字段名 长度 含义
    LSAs Headers 由需要确认的LSA的Header长度决定。 通过LSA的头部信息确认收到该LSA。

    LSA格式

    常用的LSA共有5种，分别为：Router-LSA、Network-LSA、Network-summary-LSA、ASBR-summary-LSA和AS-External-LSA。

    所有的LSA都有相同的报文头：

    ![ospf 1.png](./images/ospf 1.png)

    字段 占用空间 解释说明
    LS age 16比特 LSA产生后所经过的时间，以秒为单位。无论LSA是在链路上传送，还是保存在LSDB中，其值都会在不停的增长。
    Options 8比特 可选项：E：允许泛洪AS-External-LSA；MC：转发IP组播报文；N/P：处理Type-7 LSA；DC：处理按需链路。
    LS type 8比特 LSA的类型：Type1：Router-LSA Type2：Network-LSA Type3：Network-summary-LSA Type4：ASBR-summary-LSA Type5：AS-External-LSA Type7：NSSA-LSA
    Link State ID 32比特 与LSA中的LS Type和LSA description一起在路由域中描述一个LSA。
    Advertising Router 32比特 产生此LSA的路由器的Router ID。
    LS sequence number 32比特 LSA的序列号。其他路由器根据这个值可以判断哪个LSA是最新的。
    LS checksum 16比特 除了LS age外其它各域的校验和。
    length 16比特 LSA的总长度，包括LSA Header，以字节为单位。

    五大类👇

    Router-LSA（Type1）由路由器产生，描述了路由器的链路状态和开销，本区域内传播

    Network-LSA（Type2）由DR产生，描述本网段的链路状态，本区域内传播

    Network-summary-LSA（Type3）由ABR产生，描述区域内某个网段的路由，区域间传播（除特殊区域）

    ASBR-summary-LSA（Type4）由ABR产生，描述到ASBR的路由，非ASBR所在的区域传播（除特殊区域）

    AS-external-LSA（Type5）由ASBR产生，描述到AS外部的路由，OSPF域内传播（除特殊区域）

    NSSA LSA（Type7）由ASBR产生，描述到AS外部的路由，仅在NSSA区域内传播。

    LSA报文固定格式

    所有的LSA都有相同的报文头

    ![ospf 4.png](./images/ospf 4.png)

    字段 长度 含义
    LS age 16比特 LSA产生后所经过的时间，单位是秒。无论LSA是在链路上传输，还是保存在LSDB中，其值都会在不停的增长。
    LS type 16比特 LSA的类型：Type1：Router-LSA。Type2：Network-LSA。Type3：Inter-Area-Prefix-LSA。Type4：Inter-Area-Router-LSA。Type5：AS-external-LSA。Type7：NSSA-LSA。Type8：Link-LSA。Type9：Intra-Area-Prefix-LSA。
    Link State ID 32比特 与LS Type一起描述路由域中唯一一个LSA。
    Advertising Router 32比特 产生此LSA的ME设备的Router ID。
    LS sequence number 32比特 LSA的序列号。其他ME设备根据这个值可以判断哪个LSA是最新的。
    LS checksum 16比特 除了LS age外其它各域的校验和。
    length 16比特 LSA的总长度，包括LSA Header，以字节为单位。

    Router-LSA

    Router-LSA（Type1）：每个ME设备都会产生，描述了ME设备的链路状态和花费，在所属的区域内传播

    ![ospf 4.png](./images/ospf 4.png)

    字段 长度 解释
    Nt (NSSA translation) 1比特 如果产生此LSA的ME设备是NSSA边界路由器，则置为1，否则置为0。当该位设置为1时，ME设备无条件将NSSA-LSA转换为AS-external-LSA。
    x 1比特 废弃位。
    V（Virtual Link） 1比特 如果产生此LSA的ME设备是虚连接的端点，则置为1，否则置为0。
    E（External） 1比特 如果产生此LSA的ME设备是ASBR，则置为1，否则置为0。
    B（Border） 1比特 如果产生此LSA的ME设备是ABR，则置为1，否则置为0。
    Options 24比特 设备支持的可选能力。
    Type 8比特 设备连接的基本描述：1：点到点连接到另一台ME设备。2：连接到传输网络。3：预留位。4：虚拟链路。
    metric 16比特 链路的开销值。
    Interface ID 32比特 为接口分配的ID
    Neighbor Interface ID 32比特 邻居设备的接口ID。对于传输链路，指DR的接口ID。对于其他链路类型，指邻居设备的接口ID。
    Neighbor Router ID 32比特 邻居设备的Router ID。对于传输链路，指DR的Router ID。对于其他链路类型，指邻居设备的Router ID。

    Network-LSA

    Network-LSA（Type2）：由广播网或NBMA网络中的DR产生，Network-LSA中记录了这一网络上所有ME设备的Router ID，描述本网段的链路状态，在所属的区域内传播

    ![ospf 4.png](./images/ospf 4.png)

    字段名 长度 含义
    Options 24比特 ME设备支持的可选能力。
    Attached Router 32比特 连接在同一个网络上的所有ME设备的Router ID，也包括DR的Router ID。

    Inter-Area-Prefix-LSA

    Inter-Area-Prefix-LSA（Type3）：描述区域内某个网段的路由，由ABR产生，并通告给其他相关区域。

    ![ospf 4.png](./images/ospf 4.png)

    字段名 长度 含义
    PrefixLength 8比特 前缀长度。
    PrefixOption 8比特 前缀相关能力选项。
    Address Prefix 32比特 前缀本身的编码。

    Inter-Area-Router-LSA

    Inter-Area-Router-LSA（Type4）：由ABR产生，描述到ASBR的路由，通告给除ASBR所在区域的其他相关区域。

    ![ospf 4.png](./images/ospf 4.png)

    字段名 长度 含义
    Destination Router ID 32比特 LSA描述的ME设备的Router ID。

    AS-External-LSA

    AS-External-LSA描述到AS外部的路由，由ASBR产生。

    ![ospf 4.png](./images/ospf 4.png)

    字段 长度 含义
    E 1比特 表示外部开销类型：如果值为1，则表示Type 2外部开销类型。如果值为0，则表示Type 1外部开销类型。
    F 1比特 表示LSA中是否包括Forwarding Address：如果值为1，则表示包含。如果值为0，则表示不包含。
    T 1比特 表示LSA中是否包括External Route Tag：如果值为1，则表示包含。如果值为0，则表示不包含。
    Referenced LS Type 16比特 关联的链路状态类型。如果不为0，则表示具有这种类型的LSA和本LSA相关（参考下面的Referenced Link State ID）。
    Forwarding Address 128比特 一个完全合格的全球IPv6地址。
    External Route Tag 32比特 外部路由标识，可以用来在ASBR之间传递额外信息。
    Referenced Link State ID 32比特 关联的链路状态ID。

    NSSA-LSA

    NSSA-LSA描述到AS外部的路由，由ASBR产生。

    ![ospf 4.png](./images/ospf 4.png)

    字段 长度 含义
    E 1比特 表示外部开销类型：如果值为1，则表示Type 2外部开销类型。如果值为0，则表示Type 1外部开销类型。
    F 1比特 表示LSA中是否包括Forwarding Address：如果值为1，则表示包含。如果值为0，则表示不包含。
    T 1比特 表示LSA中是否包括External Route Tag：如果值为1，则表示包含。如果值为0，则表示不包含。
    Referenced LS Type 16比特 关联的链路状态类型。如果不为0，则表示具有这种类型的LSA和本LSA相关（参考下面的Referenced Link State ID）。
    Forwarding Address 128比特 一个完全合格的全球IPv6地址。
    External Route Tag 32比特 外部路由标识，可以用来在ASBR之间传递额外信息。
    Referenced Link State ID 32比特 关联的链路状态ID。

    Link-LSA

    每个ME设备都会为每个链路产生一个Link-LSA，描述到此Link上的link-local地址、IPv6前缀地址，并提供将会在Network-LSA中设置的链路选项，它仅在此链路内传播。

    ![ospf 4.png](./images/ospf 4.png)

    字段名 长度 含义
    Rtr Priority 8比特 接口的路由器优先级。
    Options 24比特 广播或NBMA链路上DR生成的Network-LSA中可能设置的选项。
    Link-local Interface Address 128比特 链路上始发ME设备的链路本地地址。
    Number of prefixes 32比特 LSA中IPv6地址前缀数量。

    Intra-Area-Prefix-LSA

    每个ME设备及DR都会产生一个或多个此类LSA，在所属的区域内传播。

    • ME设备产生的此类LSA，描述与Route-LSA相关联的IPv6前缀地址。

    • DR产生的此类LSA，描述与Network-LSA相关联的IPv6前缀地址。

    ![ospf 4.png](./images/ospf 4.png)

    Referenced LS Type 16比特 描述IPv6前缀地址与Router-LSA或Network-LSA相关联。如果Referenced LS Type字段是0x2001，则表示IPv6前缀地址与Router-LSA相关联。如果Referenced LS Type字段是0x2002，则表示IPv6前缀地址与Network-LSA相关联。
    Referenced Link State ID 32比特 相关联的链路状态ID。如果Referenced LS Type字段是0x2001，则Referenced Link State ID应该是0。如果Referenced LS Type字段是0x2002，则Referenced Link State ID应该是DR的接口ID。
    Referenced Advertising Router 32比特 相关联的发布路由器的ID。如果Referenced LS Type字段是0x2001，则Referenced Advertising Router字段应该是始发ME设备的Router ID。如果Referenced LS Type字段是0x2002，则Referenced Advertising Router字段应该是DR的Router ID。
